## TLDR

- Speeds up limiter-heavy `gpuSourcePort` jobs with default-on slot-accumulated safe feedback, parallel feedback for eligible unsafe chunks, smaller default GPU chunks, and asymmetric limiter-risk planning.
- Keeps decoded output matched to Tdarr's CPU-only `Normalize Audio` on the required 30s, 60s, and 30min matrix.
- This is a GPU speed release versus `1.1.6`, but it is still slower than CPU-only `Normalize Audio` on the measured matrix.

## Performance

`1.1.7` was compared against Tdarr's legacy CPU-only `Normalize Audio` plugin output. All accepted cases matched decoded audio with `energy_similarity=1.000000000`.

| Case | Legacy CPU `Normalize Audio` | GPU Normalize Audio `1.1.7` | Speed vs CPU | Decoded audio vs CPU output |
| --- | ---: | ---: | ---: | --- |
| 30s | `10.1s` | `16.6s` | `0.608x` | Passed, `energy_similarity=1.000000000`. |
| 60s | `16.7s` | `31.2s` | `0.536x` | Passed, `energy_similarity=1.000000000`. |
| 30min | `544.0s` | `841.4s` | `0.646x` | Passed, `energy_similarity=1.000000000`. |

Compared with `1.1.6`, the 30min GPU time improved from `964.1s` to `841.4s`, a `12.7%` wall-time reduction. Exact apply time on the same case improved from about `615.0s` to about `515.2s`; exact stats remained around `114.9s`.

Do not read the `Speed vs CPU` column as a faster-than-CPU claim. Values below `1.0x` mean GPU is still slower than CPU-only `Normalize Audio` for that case.

## Changes

- Adds default-on slot-accumulated safe-feedback work for qualifying 5.1 f64 exact-limiter chunks.
- Adds default-on parallel feedback accumulation for eligible unsafe chunks while preserving exact limiter decisions.
- Changes the default exact-limiter risk expansion to an asymmetric before/after shape to keep long-case parity with fewer flagged windows.
- Changes the default GPU chunk size from `64 MiB` to `32 MiB` for better long-case scheduling on the measured limiter-heavy workload.
- Keeps the normal streaming/two-pass `gpuSourcePort` behavior so long media does not create huge raw PCM bridge files.
- Continues the runtime modularization so stats, gain, apply, sync feedback, and CUDA setup are easier to audit without changing the public plugin flow.
