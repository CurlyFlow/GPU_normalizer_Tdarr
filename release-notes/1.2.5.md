## TLDR

- `1.2.5` is a parity-preserving long-media speed release over `1.2.4`.
- The full final installed-plugin default matrix passed decoded parity: `30s`, `60s`, `10min`, `30min`, and `60min`.
- Long combined jobs remain faster than Tdarr CPU `Normalize Audio`: `1.805x` on `10min`, `1.926x` on `30min`, and `1.791x` on `60min`.
- Original 5.1 GPU time improved enough to clear the release speed gate versus `1.2.4` on `10min`, `30min`, and `60min` while keeping `energy_similarity=1.000000000`.
- Codec smoke passed decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, multi-audio language-scope behavior, and no-language-match hard-fail behavior.

## Performance

`1.2.5` was compared against Tdarr's legacy CPU-only `Normalize Audio` output. The benchmark source is a 5.1 file without an existing stereo track, so the accepted GPU job outputs both the normalized original 5.1 stream and a generated normalized 2-channel fallback. Both decoded output streams were compared against CPU output and passed with `energy_similarity=1.000000000`.

`SRC_CPU` means the Tdarr CPU `Normalize Audio` reference/cache. For generated fallback, the harness creates the matching 2-channel source and runs CPU `Normalize Audio` on it. `OUR_CPU` means CPU-side work inside the GPU plugin run.

`Speed vs SRC_CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

## Changes

- Improves long-media orchestration while preserving CPU-normalizer decoded parity.
- Defaults exact stream ring mode and aligned decode prefetch on for the source-port runtime.
- Defaults original-track apply chunks to `2MiB`.
- Changes paired fallback apply defaults to use separate audio encoders plus final mux instead of paired direct mux.
- Starts first-ready paired apply streams as each stream's gain gate is ready, instead of waiting for both streams before any per-stream apply starts.
- Keeps original CPU loudnorm separate from split-stats by default and raises early CPU loudnorm prefetch to `2` measurements.
- Defaults paired apply no-tee and split-stats prefetch-during-stats on.
- Hardens split-stats completion handling when a subprocess reports signal/null exit after valid stats-cache artifacts were written.
- Keeps the `1.2.4` language defaults and safety behavior: `Normalize ONLY Languages=eng,en`, `Remove Other Languages=true`, generated first-language 2-channel fallback, and fallback to undetected audio when selected languages do not exist.

## Codec Smoke

Release codec smoke passed decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio.

The multi-audio smoke also verifies language-scope behavior:

| Scenario | Result |
| --- | --- |
| blank/all languages | Passed; all audio kept/normalized. |
| omitted default | Passed; default English selection removed other languages. |
| selected languages only | Passed; selected languages kept, others removed. |
| selected language with `Remove Other Languages=false` | Passed; the selected language was normalized and other languages copied. |
| no language match | Passed; plugin fails safely when no desired or undetected audio would remain. |

## Caveats

- Only `192k` AAC output and the tested Tdarr CPU `Normalize Audio` contract are covered by this release validation.
- Generated 2ch fallback is still slower than CPU by itself.
- Short clips can be slower or noisy because startup, stream setup, decode, and encode overhead dominate.
- Language filtering uses source metadata. Wrong or missing language tags cannot be corrected automatically.
