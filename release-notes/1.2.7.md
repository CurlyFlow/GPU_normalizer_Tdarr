## TLDR

- `1.2.7` is a parity-preserving long-media combined-wall speed release over `1.2.6`.
- The full installed-plugin matrix passed decoded parity: `60s`, `10min`, `30min`, and `60min`.
- Long combined jobs are faster than Tdarr CPU `Normalize Audio`: `2.537x` on `10min`, `2.777x` on `30min`, and `3.039x` on `60min`.
- Compared with stable `1.2.6`, combined GPU wall time improved by `22%` on `10min`, `26%` on `30min`, and `30%` on `60min` by speed ratio while keeping `energy_similarity=1.000000000`.
- Codec smoke passed decoded parity with refreshed SRC_CPU references for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, multi-audio language-scope behavior, and no-language-match hard-fail behavior.

## Performance

`1.2.7` was compared against Tdarr's legacy CPU-only `Normalize Audio` output. The benchmark source is a 5.1 file without an existing stereo track, so the accepted GPU job outputs both the normalized original 5.1 stream and a generated normalized 2-channel fallback. Both decoded output streams were compared against CPU output and passed with `energy_similarity=1.000000000`.

`SRC_CPU` means the Tdarr CPU `Normalize Audio` reference/cache. For generated fallback, the harness creates the matching 2-channel source and runs CPU `Normalize Audio` on it. `OUR_CPU` means CPU-side work inside the GPU plugin run.

`Speed vs SRC_CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

## Changes

- Runs the same-source original stream and generated stereo fallback as independent parallel serial routes on long `gpuSourcePort` jobs.
- Disables paired fallback apply by default so the original and generated fallback routes can overlap wall time instead of waiting for a combined paired runtime.
- Keeps parallel serial execution constrained to exactly one same-source original plus generated fallback pair. Multi-audio normalization remains serial to preserve language-scope and per-stream safety behavior.
- Uses CPU first-pass measurement by default for media shorter than `120s` so refreshed codec-smoke SRC_CPU references keep exact decoded parity. Long jobs keep GPU first-pass measurement for the accepted speed path.
- Keeps the `1.2.6` language defaults and safety behavior: `Normalize ONLY Languages=eng,en`, `Remove Other Languages=true`, generated first-language 2-channel fallback, and fallback to undetected audio when selected languages do not exist.

## Codec Smoke

Release codec smoke passed decoded parity with refreshed SRC_CPU references for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio.

The multi-audio smoke also verifies language-scope behavior:

| Scenario | Result |
| --- | --- |
| blank/all languages | Passed; all audio kept/normalized. |
| omitted default | Passed; default English selection removed other languages. |
| selected languages only | Passed; selected languages kept, others removed. |
| selected language with `Remove Other Languages=false` | Passed; the selected language was normalized and other languages copied. |
| no language match | Passed; plugin fails safely when no desired or undetected audio would remain. |

## Caveats

- Only `192k` AAC output and the tested Tdarr CPU `Normalize Audio` contract are covered by this release validation.
- Short media uses CPU first-pass measurement by default for parity, so the large speed gains are aimed at longer media.
- Generated 2-channel fallback can be slower by itself on some `10min`/`30min` cases, but the combined long job is faster because original and generated work overlap.
- Language filtering uses source metadata. Wrong or missing language tags cannot be corrected automatically.
