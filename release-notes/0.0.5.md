## Performance

Bug: not same as CPU normalizer. Measured against Tdarr's legacy CPU-only `Normalize Audio` plugin, pre-stable `0.0.5` was faster on short/60s smokes but failed decoded-audio acceptance.

| Case | Legacy CPU `Normalize Audio` | GPU Normalize Audio `0.0.5` | Speed vs CPU | Decoded audio vs CPU output |
| --- | ---: | ---: | ---: | --- |
| 12s 5.1 smoke, `gpuSourcePort` | `4.111s` | `2.584s` | `1.591x` | Failed: same mismatch class as `0.0.2`. |
| 60s public movie clip, `gpuSourcePort` | `17.794s` | `6.820s` | `2.609x` | Failed: `energy_similarity=0.597579`, SNR `7.906 dB`. |

This release is not performance-accepted because faster output below the CPU-normalizer audio target is a failed result.
