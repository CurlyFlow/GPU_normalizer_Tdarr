## TLDR

- Speeds up limiter-heavy `gpuSourcePort` jobs by skipping safe output-feedback accumulation after the feedback threshold is active, while still rendering and clamping decoded output normally.
- Keeps decoded output matched to Tdarr's CPU-only `Normalize Audio` on the required 30s, 60s, 10min, and 30min matrix.
- This is a GPU speed release versus `1.1.7`, but it is still slower than CPU-only `Normalize Audio` on the measured matrix.

## Performance

`1.1.8` was compared against Tdarr's legacy CPU-only `Normalize Audio` plugin output. All accepted cases matched decoded audio with `energy_similarity=1.000000000`.

| Case | Legacy CPU `Normalize Audio` | GPU Normalize Audio `1.1.8` | Speed vs CPU | Decoded audio vs CPU output |
| --- | ---: | ---: | ---: | --- |
| 30s | `10.1s` | `15.7s` | `0.641x` | Passed, `energy_similarity=1.000000000`. |
| 60s | `16.7s` | `27.5s` | `0.608x` | Passed, `energy_similarity=1.000000000`. |
| 10min | `169.8s` | `234.2s` | `0.725x` | Passed, `energy_similarity=1.000000000`. |
| 30min | `544.0s` | `712.1s` | `0.764x` | Passed, `energy_similarity=1.000000000`. |

Compared with `1.1.7`, the 30min GPU time improved from `841.4s` to `712.1s`, a `15.4%` wall-time reduction. Exact apply time on the same case improved from about `515.2s` to about `363.3s`; exact stats remained around `114.9s`.

Do not read the `Speed vs CPU` column as a faster-than-CPU claim. Values below `1.0x` mean GPU is still slower than CPU-only `Normalize Audio` for that case.

## Changes

- Enables safe output-feedback skipping by default for qualifying exact-limiter chunks after the feedback threshold is active.
- Preserves decoded output parity by still running normal limiter-buffer fill, true-peak limiter state, output write, and clipping behavior.
- Keeps the risky prefilled-output skip-fill/prefix path disabled for release because it did not preserve decoded parity.
- Keeps slot-accumulated safe feedback, parallel unsafe feedback, smaller chunks, asymmetric risk planning, and streaming/two-pass `gpuSourcePort` behavior from `1.1.7`.
- Updates the accepted benchmark matrix to include the 10min 5.1 case in addition to 30s, 60s, and 30min.
