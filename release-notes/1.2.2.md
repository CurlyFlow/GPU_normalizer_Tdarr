## TLDR

- Adds `Remove Other Languages`, default on, next to `Normalize ONLY Languages`. With a real language list, audio outside the selected languages is deleted instead of copied.
- Keeps the old copy behavior when `Remove Other Languages=false`.
- If selected languages match no audio, the output keeps non-audio streams and contains no audio streams.
- Promotes the current exact-parity speed optimizations to release defaults: paired apply/direct mux, direct FIFO decode, fused first-pass measure with split stats, async encode writer queue `1024`, and current stereo source precompute path.
- This is a feature release with current speed optimizations, not a `10%` speed-target release. Same-window `30min` testing was `4.7%` faster than `1.2.1`, with exact decoded parity preserved.

## Performance

`1.2.2` was compared against Tdarr's legacy CPU-only `Normalize Audio` output. The benchmark source is a 5.1 file without an existing stereo track, so the accepted GPU job outputs both the normalized original 5.1 stream and a generated normalized 2-channel fallback. Both decoded output streams were compared against CPU output and passed with `energy_similarity=1.000000000` and `rms_diff=0`.

`SRC_CPU` means the Tdarr CPU `Normalize Audio` reference/cache. For generated fallback, the harness creates the matching 2-channel source and runs CPU `Normalize Audio` on it. `OUR_CPU` means CPU-side work inside the GPU plugin run.

`Speed vs SRC_CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

This release did not meet the earlier `10%` 30min speed target. It is being released for the language-removal feature plus the current parity-preserving speed defaults.

### 30min

| Workload | SRC_CPU reference | GPU Normalize Audio `1.2.2` | Speed vs SRC_CPU | Decoded audio vs SRC_CPU output |
| --- | ---: | ---: | ---: | --- |
| Original 5.1 | `540.3s` | `450.5s` | `1.199x` | 5.1 passed, `energy_similarity=1.000000000`. |
| Generated 2ch | `155.5s` | `450.5s` | `0.345x` | +2ch passed, `energy_similarity=1.000000000`. |
| Combined | `695.8s` | `450.6s` | `1.544x` | Both streams passed. |

Same-window comparison on the same host measured tracked `1.2.1` at `472.7s` combined and `1.2.2` at `450.6s` combined, about `4.7%` faster in that window. The older official `1.2.1` release run was faster (`438.5s`), so do not treat `1.2.2` as a guaranteed speed win on every host/load window.

## Changes

- Adds `Remove Other Languages`, default `true`, immediately after `Normalize ONLY Languages`.
- With `Remove Other Languages=true` and a real language list, non-selected audio streams are omitted from the output.
- With `Remove Other Languages=false`, non-selected audio streams are copied unchanged, matching the old behavior.
- If a real language list matches no audio streams and removal is on, the output has no audio streams but still preserves video, subtitles, attachments, data, chapters, and metadata.
- Improves all plugin setting tooltips with consistent `Default: ...` wording and clearer language-filter behavior.
- Promotes current WIP speed defaults: paired fallback apply/direct mux/direct FIFO decode, parallel close, global AAC threads `2`, stereo fallback AAC threads `1`, fused CPU loudnorm measure during split stats, async encode write queue/memory `1024`, and stereo source precompute from channel sums.

## Codec Smoke

Release codec smoke passed exact decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio.

The multi-audio smoke also verifies language-scope behavior:

| Scenario | Result |
| --- | --- |
| blank/all languages | Passed; all audio kept/normalized. |
| omitted default | Passed; default English selection removed other languages. |
| English plus German | Passed; selected languages kept, others removed. |
| German with `Remove Other Languages=false` | Passed; German normalized and other languages copied. |
| no language match | Passed; output contained no audio streams. |

## Caveats

- This is a feature release, not a `10%` speed-target release.
- Only `192k` AAC output and the tested Tdarr CPU `Normalize Audio` contract are covered by this release validation.
- Generated 2ch fallback is still slower than CPU by itself.
- Short clips can be slower because startup, stream setup, decode, and encode overhead dominate.
- Language filtering uses source metadata. Wrong or missing language tags cannot be corrected automatically.
