## TLDR

- Adds parity-safe generated 2-channel fallback handling to the default `gpuSourcePort` streaming path, including language-priority selection for the default first generated fallback and optional all-language fallback generation.
- Matches Tdarr CPU-only `Normalize Audio` decoded output for both the original 5.1 stream and the generated 2-channel fallback on the tested matrix.
- Codec smoke passed exact decoded parity across AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio inputs.
- 60min split: original 5.1 is `1.760x` faster (`1072.6s` CPU vs `609.3s` GPU), generated 2ch is `0.478x` by itself (`297.2s` CPU vs `621.7s` GPU), and the combined job is `1.107x` faster (`1369.8s` CPU vs `1237.2s` GPU).

## Performance

`1.1.11` was compared against Tdarr's legacy CPU-only `Normalize Audio` output. The benchmark source is a 5.1 file without an existing stereo track, so the accepted GPU job outputs both the normalized original 5.1 stream and a generated normalized 2-channel fallback. Both decoded output streams were compared against CPU output and passed with `energy_similarity=1.000000000`.

`Speed vs CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

## Changes

- Fixes generated-stereo source feedback parity in threaded skip paths by retaining the `ctypes` source-sum pointer and source-mode flag objects for CUDA launch argument lifetime.
- Uses exact source precompute for generated stereo fallback so CPU-compatible source window, 4-window, and short-window sums are available to threaded feedback/apply paths without recomputing the full source ring in every apply step.
- Parallelizes generated-stereo source precompute by 100ms window and routes source-safe chunks through the same skip-safe feedback framework as the main 5.1 path when exact precomputed source sums are available.
- Skips expensive generated-stereo output-feedback bookkeeping after the feedback threshold is locked; output feedback no longer affects future gain decisions in those chunks, while input feedback, limiter state, output rendering, and final flush remain exact.
- Adds `Track Order`, default `eng,en`, so generated 2-channel fallback source selection can prefer specific languages before falling back to source order.
- Adds `Only Add 2-Channel For First Language`, default `true`, so the default created-stereo behavior generates one normalized 2-channel fallback from the first non-stereo stream after language-priority sorting.
- Keeps the product path streaming through `gpuSourcePort`: FFmpeg decode streams into the GPU runtime, and GPU f64 output streams back into FFmpeg AAC encode without the old full raw PCM bridge for normal jobs.
- Preserves source channel layout for original streams, including `5.1(side)` codec cases, and keeps FFmpeg-compatible linear-mode handling for streams that CPU `Normalize Audio` treats as linear.

## Caveats

- Only `192k` AAC output and the tested Tdarr CPU `Normalize Audio` contract are covered by this release matrix.
- Codec smoke passed exact decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and a multi-audio sample using per-stream CPU `Normalize Audio` references for the multi-audio case.
- Short clips can be slower than CPU because startup, stream setup, and FFmpeg overhead dominate.
- Disable `Only Add 2-Channel For First Language` only if you really want a generated 2-channel fallback for every non-stereo audio stream/language; that adds more work and more output audio streams.
