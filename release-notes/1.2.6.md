## TLDR

- `1.2.6` is a parity-preserving long-media speed release over `1.2.5`.
- The full installed-plugin matrix passed decoded parity: `60s`, `10min`, `30min`, and `60min`.
- Long combined jobs are faster than Tdarr CPU `Normalize Audio`: `2.086x` on `10min`, `2.297x` on `30min`, and `2.199x` on `60min`.
- Compared with stable `1.2.5`, combined GPU wall time improved by `13.5%` on `10min`, `16.1%` on `30min`, and `18.5%` on `60min` while keeping `energy_similarity=1.000000000`.
- Codec smoke passed decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, multi-audio language-scope behavior, and no-language-match hard-fail behavior.

## Performance

`1.2.6` was compared against Tdarr's legacy CPU-only `Normalize Audio` output. The benchmark source is a 5.1 file without an existing stereo track, so the accepted GPU job outputs both the normalized original 5.1 stream and a generated normalized 2-channel fallback. Both decoded output streams were compared against CPU output and passed with `energy_similarity=1.000000000`.

`SRC_CPU` means the Tdarr CPU `Normalize Audio` reference/cache. For generated fallback, the harness creates the matching 2-channel source and runs CPU `Normalize Audio` on it. `OUR_CPU` means CPU-side work inside the GPU plugin run.

`Speed vs SRC_CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

## Changes

- Adds original-only GPU first-pass measurement for the original stream while keeping generated stereo fallback on the CPU loudnorm first pass for parity.
- Adds a native host output-metric path for first-pass JSON so first-pass output window sums no longer use the slow single-thread CUDA output-sums scan.
- Uses mixed source/output first-pass measurement for the original stream to keep FFmpeg-compatible input metrics while avoiding the full CPU loudnorm wait.
- Fixes a CUDA safe-feedback skip-kernel argument mismatch introduced during output-metric work; source-exact stereo fallback now receives the correct mode flag again.
- Keeps source-exact stereo fallback and paired apply defaults for the accepted release path.
- Keeps the `1.2.5` language defaults and safety behavior: `Normalize ONLY Languages=eng,en`, `Remove Other Languages=true`, generated first-language 2-channel fallback, and fallback to undetected audio when selected languages do not exist.

## Codec Smoke

Release codec smoke passed decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio.

The multi-audio smoke also verifies language-scope behavior:

| Scenario | Result |
| --- | --- |
| blank/all languages | Passed; all audio kept/normalized. |
| omitted default | Passed; default English selection removed other languages. |
| selected languages only | Passed; selected languages kept, others removed. |
| selected language with `Remove Other Languages=false` | Passed; the selected language was normalized and other languages copied. |
| no language match | Passed; plugin fails safely when no desired or undetected audio would remain. |

## Caveats

- Only `192k` AAC output and the tested Tdarr CPU `Normalize Audio` contract are covered by this release validation.
- Short clips can be slower or noisy because startup, stream setup, decode, and encode overhead dominate.
- Language filtering uses source metadata. Wrong or missing language tags cannot be corrected automatically.
