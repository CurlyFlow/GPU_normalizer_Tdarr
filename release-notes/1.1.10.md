## TLDR

- Speeds up limiter-heavy `gpuSourcePort` jobs with smaller streaming chunks, default-on stats decode prefetch, and an unsafe-window feedback skip that only activates after output feedback can no longer affect future gain decisions.
- Keeps decoded output matched to Tdarr's CPU-only `Normalize Audio` on the required 30s, 60s, 10min, and 30min matrix.
- This is a GPU speed release versus `1.1.9`; all required cases are faster than CPU-only `Normalize Audio` in the measured matrix.

## Performance

`1.1.10` was compared against Tdarr's legacy CPU-only `Normalize Audio` plugin output. All accepted cases matched decoded audio with `energy_similarity=1.000000000`.

`Speed vs CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

| Case | Legacy CPU `Normalize Audio` | GPU Normalize Audio `1.1.10` | Speed vs CPU | Decoded audio vs CPU output |
| --- | ---: | ---: | ---: | --- |
| 30s | `10.1s` | `9.4s` | `1.075x` | Passed, `energy_similarity=1.000000000`. |
| 60s | `17.5s` | `15.2s` | `1.156x` | Passed, `energy_similarity=1.000000000`. |
| 10min | `169.8s` | `114.9s` | `1.477x` | Passed, `energy_similarity=1.000000000`. |
| 30min | `544.0s` | `334.0s` | `1.629x` | Passed, `energy_similarity=1.000000000`. |

Compared with `1.1.9`, the 30min GPU time improved from `403.0s` to `334.0s`, a `17.1%` wall-time reduction. By the release table's `previous time / current time - 1` calculation, that is about `20.7%` faster. Exact stats time on the same case improved from about `114.9s` to about `108.5s`; exact apply time improved from about `112.8s` to about `97.9s`.

Strict created-stereo diagnostics also passed against a streaming CPU 2-channel `loudnorm` golden: 60s GPU `31.1s`, 10min GPU `240.5s`, both with `energy_similarity=1.000000000`.

## Changes

- Changes the default GPU chunk size from `32 MiB` to `1 MiB` for better streaming overlap on the exact limiter path.
- Enables bounded stats decode prefetch by default so FFmpeg decode can stay ahead of CUDA stats processing without materializing raw PCM bridge files.
- Skips output K-weight feedback accumulation for unsafe windows after the output feedback threshold is already locked; true-peak limiter state, limited output, input feedback, gain state, and final flush behavior are still rendered normally.
- Generalizes several 2-channel exact paths used by the optional created-stereo track so 2-channel fallback parity and performance are covered by diagnostics.
- Keeps prefix channel stats, reduced limiter risk expansion, and apply decode prefetch disabled because they were not long-matrix parity-safe or did not improve wall time.
