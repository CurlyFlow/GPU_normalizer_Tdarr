## TLDR

- `1.2.3` is a parity-preserving speed and maintainability release over `1.2.2`.
- The full installed-plugin matrix passed decoded parity: `30s`, `60s`, `10min`, `30min`, and `60min`.
- Long combined jobs are faster than Tdarr CPU `Normalize Audio`: `1.515x` on `10min`, `1.717x` on `30min`, and `1.644x` on `60min`.
- The `30min` GPU wall time improved from `450.6s` in `1.2.2` to `405.3s` in `1.2.3`, about a `10%` wall-time reduction in the same benchmark contract.
- Codec smoke still passes exact decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio language-scope behavior.

## Performance

`1.2.3` was compared against Tdarr's legacy CPU-only `Normalize Audio` output. The benchmark source is a 5.1 file without an existing stereo track, so the accepted GPU job outputs both the normalized original 5.1 stream and a generated normalized 2-channel fallback. Both decoded output streams were compared against CPU output and passed with `energy_similarity=1.000000000` and `rms_diff=0`.

`SRC_CPU` means the Tdarr CPU `Normalize Audio` reference/cache. For generated fallback, the harness creates the matching 2-channel source and runs CPU `Normalize Audio` on it. `OUR_CPU` means CPU-side work inside the GPU plugin run.

`Speed vs SRC_CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

### 60min

| Workload | SRC_CPU reference | GPU Normalize Audio `1.2.3` | Speed vs SRC_CPU | Decoded audio vs SRC_CPU output |
| --- | ---: | ---: | ---: | --- |
| Original 5.1 | `1027.3s` | `802.1s` | `1.281x` | 5.1 passed, `energy_similarity=1.000000000`. |
| Generated 2ch | `291.2s` | `802.1s` | `0.363x` | +2ch passed, `energy_similarity=1.000000000`. |
| Combined | `1318.5s` | `802.1s` | `1.644x` | Both streams passed. |

## Changes

- Keeps the `1.2.2` language behavior: `Normalize ONLY Languages` defaults to `eng,en`, `Remove Other Languages=true`, and disabling removal copies non-selected audio unchanged.
- Preserves the streaming `gpuSourcePort` path and generated 2-channel fallback behavior.
- Promotes the post-review structural cleanup that removes broad context flattening, makes split-stats and paired-apply plans explicit, validates paired-apply JSON plans, and centralizes runtime environment parsing.
- Improves worker/process cleanup by using process-group termination for shell-launched FFmpeg/runtime work and fail-fast cancellation for parallel stream workers.
- Adds release snapshot manifest validation so packaging checks the tracked release folder against generated metadata before building the zip.

## Codec Smoke

Release codec smoke passed exact decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio.

The multi-audio smoke also verifies language-scope behavior:

| Scenario | Result |
| --- | --- |
| blank/all languages | Passed; all audio kept/normalized. |
| omitted default | Passed; default English selection removed other languages. |
| selected languages only | Passed; selected languages kept, others removed. |
| selected language with `Remove Other Languages=false` | Passed; the selected language was normalized and other languages copied. |
| no language match | Passed; output contained no audio streams. |

## Caveats

- Only `192k` AAC output and the tested Tdarr CPU `Normalize Audio` contract are covered by this release validation.
- Generated 2ch fallback is still slower than CPU by itself.
- Short clips can be slower because startup, stream setup, decode, and encode overhead dominate.
- Language filtering uses source metadata. Wrong or missing language tags cannot be corrected automatically.
