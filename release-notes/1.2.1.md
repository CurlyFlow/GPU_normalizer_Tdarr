## TLDR

- Adds `Normalize ONLY Languages`: default UI value is `eng, en`; clearing the field normalizes all languages; no-match scopes pass the file through unchanged without taking the GPU lock or muxing.
- Cleans up default job logs so normal jobs show input audio, settings, plan, decisions, and final result; detailed `profile_stage`/FFmpeg output is behind `Debug Logging`.
- Removes stale hidden flow inputs `gpuChunkMiB` and `maxPcmMiB`.
- Required matrix and codec smoke passed exact decoded parity. 60min combined job improved from `894.2s` in `1.2.0` to `861.9s` in `1.2.1` while preserving decoded parity.

## Performance

`1.2.1` was compared against Tdarr's legacy CPU-only `Normalize Audio` output. The benchmark source is a 5.1 file without an existing stereo track, so the accepted GPU job outputs both the normalized original 5.1 stream and a generated normalized 2-channel fallback. Both decoded output streams were compared against CPU output and passed with `energy_similarity=1.000000000` and `rms_diff=0`.

`SRC_CPU` means the Tdarr CPU `Normalize Audio` reference/cache. For generated fallback, the harness creates the matching 2-channel source and runs CPU `Normalize Audio` on it. `OUR_CPU` means CPU-side work inside the GPU plugin run.

`Speed vs SRC_CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

## Changes

- Adds `Normalize ONLY Languages`. The release/UI default is `eng, en`; clear the field or enter `all` to normalize all audio languages.
- If `Normalize ONLY Languages` matches no audio streams, the plugin now returns the original file unchanged without GPU lock acquisition, mux, verify, publish, or cleanup work.
- Reworks default job logs into readable input/settings/plan/result messages. Detailed runtime profile lines and successful FFmpeg output require `Debug Logging=true`.
- Removes hidden flow-input plumbing for `gpuChunkMiB` and `maxPcmMiB`; runtime paths use internal defaults and streaming mode no longer exposes a stale PCM guard.
- Fixes a WIP regression where removing the PCM guard accidentally removed `cfg.mib` from runtime config.

## Caveats

- Only `192k` AAC output and the tested Tdarr CPU `Normalize Audio` contract are covered by this release matrix.
- Generated 2ch fallback is still slower than CPU by itself; the combined long job wins because original 5.1 normalization and overlapped CPU/GPU work dominate over long durations.
- Short clips can still be slower or near-neutral because startup, stream setup, and FFmpeg overhead dominate.
- Release codec smoke passed exact decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio.
- Language filtering and track ordering use source metadata. Wrong or missing language tags cannot be corrected automatically.
