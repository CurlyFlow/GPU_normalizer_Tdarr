## TLDR

- Modularizes the `gpuSourcePort` runtime into smaller Python modules while keeping the loudnorm planner/apply behavior unchanged.
- Keeps decoded output matched to Tdarr's CPU-only `Normalize Audio` on the required 30s, 60s, and 30min matrix.
- This is a maintainability release, not a speed release; limiter-active media is still slower than CPU normalize.

## Performance

`1.1.5` was compared against Tdarr's legacy CPU-only `Normalize Audio` plugin output. All accepted cases matched decoded audio with `energy_similarity=1.000000000`.

| Case | Legacy CPU `Normalize Audio` | GPU Normalize Audio `1.1.5` | Speed vs CPU | Decoded audio vs CPU output |
| --- | ---: | ---: | ---: | --- |
| 30s | `10.1s` | `24.9s` | `0.406x` | Passed, `energy_similarity=1.000000000`. |
| 60s | `16.7s` | `50.1s` | `0.334x` | Passed, `energy_similarity=1.000000000`. |
| 30min | `544.0s` | `1513.4s` | `0.359x` | Passed, `energy_similarity=1.000000000`. |

Compared with `1.1.4`, performance is effectively unchanged within normal run noise. The 30min case moved from `1507.6s` to `1513.4s`, with exact f64 stats still around `114.9s` and exact apply around `1158.5s`.

## Changes

- Moves the embedded fallback PTX into its own runtime module while keeping the external `.ptx` file as the normal path.
- Extracts runtime argument/config derivation, stream process helpers, CUDA kernel lookup, CUDA buffer allocation/free, K-weighted prefix-state calculation, and prelimiter risk-map calculation.
- Updates release/staging/package manifests so all new runtime modules are included in deployed and packaged plugin folders.
- Keeps streaming/two-pass behavior so long media does not create huge raw PCM bridge files.
