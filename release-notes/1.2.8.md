## TLDR

- `1.2.8` is a parity-preserving long-media speed and stability release over `1.2.7`.
- The installed-plugin `60s` smoke and release codec smoke passed decoded parity on the final `1.2.8` snapshot.
- Long combined jobs are faster than Tdarr CPU `Normalize Audio`: `3.073x` on `10min`, `3.298x` on `30min`, and `3.522x` on `60min` in the accepted candidate matrix.
- Compared with stable `1.2.7`, combined GPU wall time improved by `17%` on `10min`, `16%` on `30min`, and `14%` on `60min` while keeping `energy_similarity=1.000000000`.
- Fixes a post-success stereo-fallback runtime teardown crash that could mark an otherwise completed generated 2-channel fallback as failed.
- Fixes a source-rate split-stats ingress mismatch that could truncate AC3 5.1 original-stream stats/apply work during codec validation.

## Performance

`1.2.8` was compared against Tdarr's legacy CPU-only `Normalize Audio` output. The benchmark source is a 5.1 file without an existing stereo track, so the accepted GPU job outputs both the normalized original 5.1 stream and a generated normalized 2-channel fallback. Both decoded output streams were compared against CPU output and passed with `energy_similarity=1.000000000` in the accepted candidate matrix.

`SRC_CPU` means the Tdarr CPU `Normalize Audio` reference/cache. For generated fallback, the harness creates the matching 2-channel source and runs CPU `Normalize Audio` on it. `OUR_CPU` means CPU-side work inside the GPU plugin run.

`Speed vs SRC_CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

## Changes

- Promotes the accepted long-media default stack from the `1.2.8-wip` optimization loop.
- Starts mixed GPU first-pass work in parallel and applies the accepted original-stream first-pass offset bias for long media.
- Keeps the same-source original stream and generated stereo fallback on parallel serial routes with the accepted fallback start delay.
- Uses the accepted direct FIFO/decode-first subpath when paired fallback apply is explicitly enabled, while keeping paired fallback apply itself disabled by default.
- Enables the accepted AAC fast coder path for the release default.
- Keeps rejected/no-promote probes disabled, including AAC disable-tools, streaming AAC fast, source-rate original processing, original encode f32/sample-rate overrides, direct-mux variants, event writer, cache slimming, split-stats scheduling variants, risk-window changes, and async/pinned/writev/OS-write probes.
- Hardens successful GPU runtime exit so a post-summary CUDA/Python teardown fault does not turn a completed stereo fallback into a failed job.
- Keeps stats and first-pass runtime decode plans at the decode command's output rate, avoiding incorrect ingress resampling on source-rate reruns.

## Codec Smoke

Release codec smoke passed on the final `1.2.8` snapshot and covers AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio.

All codec-smoke cases passed decoded parity with `energy_similarity=1.000000000`.

The multi-audio smoke also verifies language-scope behavior:

| Scenario | Result |
| --- | --- |
| blank/all languages | Must pass; all audio kept/normalized. |
| omitted default | Must pass; default English selection removes other languages. |
| selected languages only | Must pass; selected languages kept, others removed. |
| selected language with `Remove Other Languages=false` | Must pass; the selected language is normalized and other languages copied. |
| no language match | Must pass; plugin fails safely when no desired or undetected audio would remain. |

## Caveats

- Only `192k` AAC output and the tested Tdarr CPU `Normalize Audio` contract are covered by this release validation.
- Short media remains a smoke/parity probe, not public speed evidence.
- Language filtering uses source metadata. Wrong or missing language tags cannot be corrected automatically.
