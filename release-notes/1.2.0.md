## TLDR

- Fixes generated 2-channel fallback scope: with the default first-language setting, an existing stereo track in another language no longer suppresses the generated stereo track for the prioritized language.
- Generated stereo now chooses the best same-language source automatically, preferring 5.1/6-channel over 7.1/8-channel when a fallback must be created.
- Full required matrix passed exact decoded parity for both streams: original 5.1 and generated normalized 2-channel fallback.
- 60min split: original 5.1 is `2.241x` faster (`1027.3s` SRC_CPU vs `458.3s` GPU), generated 2ch is `0.674x` by itself (`291.2s` SRC_CPU vs `432.0s` GPU), and the combined job is `1.474x` faster (`1318.5s` SRC_CPU vs `894.2s` GPU).

## Performance

`1.2.0` was compared against Tdarr's legacy CPU-only `Normalize Audio` output. The benchmark source is a 5.1 file without an existing stereo track, so the accepted GPU job outputs both the normalized original 5.1 stream and a generated normalized 2-channel fallback. Both decoded output streams were compared against CPU output and passed with `energy_similarity=1.000000000` and `rms_diff=0`.

`SRC_CPU` means the Tdarr CPU `Normalize Audio` reference/cache. For generated fallback, the harness creates the matching 2-channel source and runs CPU `Normalize Audio` on it. `OUR_CPU` means CPU-side work inside the GPU plugin run.

`Speed vs SRC_CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

## Changes

- Makes generated 2-channel fallback language-aware. Default first-language scope now ensures the first `Track Order` language has a stereo track; a stereo track in another language does not satisfy that requirement.
- Chooses the generated-stereo source automatically within the selected language, preferring 5.1/6-channel over 7.1/8-channel sources.
- Keeps the all-languages mode: disable `Only Generate 2-Channel For First Language` to create generated stereo for every non-stereo audio stream.
- Updates plugin UI wording to make generated stereo behavior explicit.
- Adds `BEFORE`, `PLAN`, and `AFTER` audio summaries to the benchmark report so stream ordering and generated-track decisions are visible.

## Caveats

- Only `192k` AAC output and the tested Tdarr CPU `Normalize Audio` contract are covered by this release matrix.
- Generated 2ch fallback is still slower than CPU by itself; the combined long job wins because original 5.1 normalization and overlapped CPU/GPU work dominate over long durations.
- Short clips can still be slower or near-neutral because startup, stream setup, and FFmpeg overhead dominate.
- Release codec smoke passed exact decoded parity for AAC stereo, MP3 stereo, AC3 5.1, E-AC-3 5.1, DTS 5.1, and multi-audio.
- Language ordering uses source metadata. Wrong or missing language tags cannot be corrected automatically.
