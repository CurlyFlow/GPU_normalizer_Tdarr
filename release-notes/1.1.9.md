## TLDR

- Speeds up limiter-heavy `gpuSourcePort` jobs by parallelizing safe feedback-skip apply windows while keeping exact state updates and normal limiter/output rendering.
- Keeps decoded output matched to Tdarr's CPU-only `Normalize Audio` on the required 30s, 60s, 10min, and 30min matrix.
- This is a GPU speed release versus `1.1.8`; the measured 10min and 30min cases are faster than CPU-only `Normalize Audio`, while 30s and 60s are still slower.

## Performance

`1.1.9` was compared against Tdarr's legacy CPU-only `Normalize Audio` plugin output. All accepted cases matched decoded audio with `energy_similarity=1.000000000`.

`Speed vs CPU` above `1.0x` means the GPU plugin was faster than CPU for that case. Values below `1.0x` mean the GPU plugin was slower.

| Case | Legacy CPU `Normalize Audio` | GPU Normalize Audio `1.1.9` | Speed vs CPU | Decoded audio vs CPU output |
| --- | ---: | ---: | ---: | --- |
| 30s | `10.1s` | `13.9s` | `0.724x` | Passed, `energy_similarity=1.000000000`. |
| 60s | `16.7s` | `20.0s` | `0.834x` | Passed, `energy_similarity=1.000000000`. |
| 10min | `169.8s` | `139.4s` | `1.218x` | Passed, `energy_similarity=1.000000000`. |
| 30min | `544.0s` | `403.0s` | `1.350x` | Passed, `energy_similarity=1.000000000`. |

Compared with `1.1.8`, the 30min GPU time improved from `712.1s` to `403.0s`, a `43.4%` wall-time reduction. Exact apply time on the same case improved from about `363.3s` to about `112.8s`; exact stats remained around `114.9s`.

## Changes

- Adds a default-on parallel safe feedback-skip apply path for qualifying 5.1 f64 exact-limiter chunks.
- Preserves decoded output parity by keeping exact sequential gain, feedback, limiter-ring, and state transitions while using CUDA threads for limiter-buffer fill and current output clamp/write inside safe windows.
- Keeps the risky prefilled-output skip-fill/prefix shortcut disabled because it did not preserve decoded parity.
- Adds runtime profile counters for parallel safe feedback-skip chunks and apply time.
- Keeps slot-accumulated safe feedback, parallel unsafe feedback, smaller chunks, asymmetric risk planning, and streaming/two-pass `gpuSourcePort` behavior from `1.1.8`.
